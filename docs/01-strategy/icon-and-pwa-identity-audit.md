# Icon and PWA Identity Audit

**Date:** 2026-05-14  
**Scope:** Favicon, apple-touch-icon, theme-color across AREI web surfaces  
**Status:** Implemented

---

## Summary verdict

All three AREI web surfaces were missing `apple-touch-icon` and `theme-color`, causing iOS to generate generic dark fallback icons when users add sites to their home screen. AREI landing additionally used an inline data-URI favicon which iOS cannot reliably use for saved home screen icons. AREI admin had no favicon at all. Fixes implemented in this pass — no page layout, copy, or product logic was changed.

---

## Root cause

1. **iOS ignores data-URI favicons** for Add to Home Screen icons — it requires a real file URL. AREI landing's `data:image/svg+xml,...` href was silently discarded by iOS, producing a dark screenshot fallback.
2. **`apple-touch-icon` is the authoritative iOS home screen icon source.** Without it, iOS takes a low-quality page screenshot. Modern iOS (15+) can fall back to `favicon.svg` for a clean icon, but only if the favicon is a real file URL with a solid background.
3. **AREI admin had no favicon at all**, so every browser and OS showed a generic placeholder.

## PNG generation

`apple-touch-icon.png` files (180×180) were generated by [`scripts/gen-icons.js`](../../scripts/gen-icons.js) — a pure Node.js script using only the built-in `zlib` module. No new dependencies added. Re-run with:

```sh
node scripts/gen-icons.js
```

## Future improvements

- **`manifest.webmanifest`** for each surface — enables proper PWA install prompt, standalone display mode, and richer home screen metadata (name, short_name, display, background_color). Straightforward to add per-app once icon set is stable.
- **SVG mark** in favicon — the current icons are solid-color rects; adding a minimal AREI wordmark or geometric mark would improve recognisability at small sizes. Requires finalising the mark from `docs/brand/source/`.
- **Maskable icon variant** — for Android adaptive icons, a separate `icon-maskable.png` with safe-zone padding is needed. Low priority until Android usage is tracked.
- **Precomposed apple-touch-icon** — adding `rel="apple-touch-icon-precomposed"` prevents iOS from adding gloss effects on older iOS versions. Not critical for current iOS 15+ target.
