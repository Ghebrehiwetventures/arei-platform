## A · Canonical brand state (from brand guidelines v1.0)

### A.1 Brand architecture
> AREI is the master brand and methodology owner. Each country market is a member of the same family, distinguished only by the country name on line 1 of the lockup. There are no sub-brands beyond this rule.
>
> **KazaVerde — consumer-facing legacy mark, sunsetting. Not part of the core AREI identity system. Reference only; do not use in new collateral.** *(manual-01-foundations.jsx, §03)*

### A.2 Canonical color palette (8 tokens)

| Name | Hex | Role | Contrast |
|---|---|---|---|
| Ink | `#0A0A0A` | Primary text, **master surfaces**, KPI numbers | AAA on paper |
| Paper | `#FFFFFF` | Default surface — cards, reports, web body | Pair with ink |
| Bone | `#FAFAFA` | Page ground — slightly warmer than pure white | Pair with ink |
| Off-white | `#F2F0EC` | Secondary surface — sidebars, alt rows | Pair with ink |
| Sage | `#8ECFBF` | Brand accent **on ink** — ticker glyphs, on-dark mark | AA on ink |
| Sage deep | `#2D4A42` | Editorial dark — chart highlights, secondary surfaces, **positive deltas** | AA on paper |
| Gray 500 | `#888888` | Captions, eyebrow labels, table dividers | Body neutral |
| Coral | `#C44A3A` | Negative deltas, do-not stamps. **Paper only, never as hero.** | AA on paper |

### A.3 Pairing rules (§06.2)
- **Master surfaces:** ink ground + sage mark + sage deep secondary (ticker rows, hero modules, dark report covers)
- **Editorial / paper:** paper or bone ground + ink type + off-white as alt row (reports, web body, most documents)
- **Data delta:** sage deep for positive, coral for negative. Never red-on-green next to each other.

### A.4 Forbidden uses (§06.3)
- Country flag palettes (never nationalist colors for country indices)
- Gradients (solid color only)
- **High-saturation accents** (no magenta, electric blue, lime)
- Coral on ink (paper only, never hero)

### A.6 Chart canon (dataviz.jsx — IndexChart)
- Time-series with hairline axes, mono labels, tabular-nums
- **Filled area = `--kv-green-tint` (very pale sage)**
- **Line = ink `#0a0a0a` 1.5px stroke, round joins**
- **Last marker = ink-filled circle r=4 + outer ring r=9 opacity .3**
- Dashed baseline emphasis at 100 (rule color)
- 100 = baseline (Jan 2024)
- Y-grid hairlines + axis text in gray-500

## C · Divergences & proposed reconciliation

Ranked by impact. Each gets a separate PR (or grouped into 2-3 PRs).

### C.1 [HIGH] arei-landing: positive delta to sage deep, fix off-palette greens

**What:** Brand canonical says positive delta = sage deep `#2D4A42`. We're using `#2e7d52`. Also LIVE-dot and Market 01 ACTIVE use `#22c55e` (bright/lime, explicitly forbidden by brand "no high-sat lime").

**Action:**
- `.xfm-delta-up` color: `#2e7d52` → `#2d4a42` (sage deep)
- `.ey-dot` background: `#22c55e` → `#8ECFBF` (sage, per "ticker glyphs" + "status indicator" canonical use)
- `--bp-active` (light): `#2d6a4f` → `#2d4a42` (sage deep, since used for chart hero bar / bench value)
- `--bp-status` (light): `#22c55e` → `#8ECFBF` (sage for active node indicator)

**Risk:** Low. Visual change is subtle (still green-ish for positive). Brings landing into strict palette compliance.

### C.2 [HIGH] arei-landing: hero CTA button back to sage

**What:** Brand canonical master surface pairing = "ink ground + **sage mark** + sage deep secondary". Hero is master surface. Hero CTA button is currently white-on-ink. Per brand it should be sage-on-ink to match the master-mark.

**Action:** `.hero-section .btn-primary` background: `#fafafa` → `#8ECFBF` (sage), border same, text `#0a0a0a` (ink). Matches mark on ink.

**Risk:** Low. Visually stronger CTA, on-brand.

### C.3 [HIGH] arei-landing: Net Urban Population chart redesign

**What:** Brand canonical chart spec (dataviz.jsx IndexChart) is time-series with sage-tint area + ink line + filled circle marker. Our current implementation is horizontal bars with Africa highlighted.

**Action:** Replace bar chart with time-series chart showing 2020 → 2050 urban population growth per region (Africa, Asia, Latin America, North America, Europe). Africa series gets sage-deep fill or distinctive treatment; others ink at varying opacity. Hairline grid, mono labels.

**Risk:** Medium. Requires new data layout and reasonable projection display. Worth its own PR.

### C.4 [MED] docs: align all three doc files with current canonical

**What:**
- `docs/brand/README.md` has wrong positive-delta color and ambiguous "Sans for prose" guidance.
- `docs/04-platform/visual-identity.md` is internally contradictory ("one unified accent" is under "avoid" but README says all share sage).
- `docs/04-platform/design-tokens-discovery.md` is stale (pre-brand-pass world).

**Action:**
- README: change semantic positive-green to sage-deep `#2D4A42`. Clarify Sans-for-prose scope (where "prose" applies).
- visual-identity.md: fix contradiction. Remove "one unified accent" from §8 avoid-list since family-shared sage IS the rule.
- design-tokens-discovery.md: mark as superseded; either delete or rewrite as "current state" snapshot (May 2026).

**Risk:** Doc-only. Low.

### C.5 [MED] kazaverde-web: add missing data-series tokens

**What:** Brand canonical foundations.jsx defines `--kv-accent-amber` `#cfb48e` and soft data-series coral `#cf8e8e`. KazaVerde globals.css doesn't define these.

**Action:** Add the two missing tokens to `globals.css`. Used in chart series 4 (pending) and series 5 (alert).

**Risk:** Additive. Low.

### C.6 [MED] kazaverde-web: clean up legacy tokens

**What:** Lines 109-137 of `globals.css` have explicit "LEGACY" comment with chocolate/lime hsl tokens (`--bg`, `--ac`, `--gr`, etc.) that exist only so archived routes parse. With Cape Verde rename and KazaVerde sunset, time to verify these are truly unused and remove.

**Action:** Audit references to legacy tokens. If unused, remove. If used, migrate.

**Risk:** Low if audit confirms unused; medium if surprise references found.

### C.7 [LOW] arei-admin: harmonize bone background to canonical

**What:** Admin uses `#f7f6f2` for background, canonical bone is `#FAFAFA`. Discovery doc accepts this drift but it's still drift.

**Action:** Decision call — keep drift (admin is more "warm-cream" than landing/KV, intentional density difference) OR snap to canonical bone.

**Risk:** Low. Mostly subjective.

### C.8 [LOW] arei-landing: footer-internal dividers

**What:** Brand says dividers "suggest, not enforce". `.footer-col h4` has `border-bottom`, `.footer-bottom` has `border-top`. Visible dividers.

**Action:** Remove these two dividers; let typography hierarchy and whitespace carry the separation.

**Risk:** Very low. Cosmetic.

### C.9 [LOW] arei-landing: surface card icon block height

**What:** Icon blocks 136px tall, feel oversized for the content underneath.

**Action:** Reduce to 96–100px.

**Risk:** Low.

### C.10 [BIG · LATER] Token namespace migration: `--kv-*` → `--arei-*`

**What:** Brand says KazaVerde is sunsetting but tokens still use `--kv-*` namespace. Long-term, namespace should reflect the master brand (AREI).

**Action:** Phased rename. Big cross-cutting change. Defer until brand-guidelines v2 (which presumably will codify the rename).

**Risk:** High if done eagerly. Park for now.

## E · Open questions for decision

1. **Sans-for-prose scope.** Brand canonical says sans is "for editorial long-form — articles, briefings, essays. Anything more than two paragraphs of mono reads cold." On AREI landing, no paragraph exceeds 4 short sentences. Therefore mono-only is correct. On Cape Verde, articles likely qualify as long-form → sans should be in use. Q: should we verify Cape Verde article body actually uses sans, and if not, add it?

2. **Status-indicator color (LIVE dot, Market 01 ACTIVE).** Brand canonical doesn't explicitly specify a status-green; it uses sage for "ticker glyphs" and "on-dark mark". Does sage `#8ECFBF` read clearly enough as "live/active" on light backgrounds? Or do we accept a small green deviation for status semantics (and codify it in brand v2)?

3. **arei-landing hero CTA color.** Currently white on ink (PR #167 final). Brand says master surface = ink + sage + sage-deep. Should hero CTA flip back to sage? My answer: yes (see C.2).

4. **KazaVerde token namespace future.** Brand says KazaVerde is sunsetting. Tokens still `--kv-*`. When does the rename happen — brand v2 release, or sooner as a foundational refactor?
