# AREI · Voice & Language

**Status:** Canonical · applies to all AREI surfaces and materials.
**Scope:** Landing page, investor brief, admin Strategy Check, broker materials, index publications, briefings.

---

## Voice principle

AREI speaks like an institutional data company with a founder's thesis: calm, precise, direct, and readable. Clear before clever. Specific before abstract. No hype, no broker language, no SaaS fluff.

The same voice applies across every surface. A sentence that reads well on the landing page should read well in the investor brief. A disclosure line that belongs in a briefing belongs in the same tone as the index methodology. One company, one register.

---

## Writing rules

**1. Short sentences first.**
If a sentence has two thoughts, consider splitting it. If a clause is doing extra work that the reader doesn't need, cut it.

**2. Strong thesis lines.**
State the position, then support it. Not: "One of the challenges facing African property markets is the fragmented nature of available data." Instead: "The data layer is missing."

**3. Concrete before abstract.**
"17 sources mapped in Cape Verde" beats "significant source coverage across the first market." Name the number. Name the market.

**4. No methodology stacking.**
Do not pile technical terms into a single phrase. "Scored, source-linked, methodology-disclosed, published signal" is four adjectives for one noun. Pick the one that carries the meaning. Often it is just: "published market signal."

**5. Active voice. Present tense where possible.**
"AREI turns raw listings into indexed records" beats "listings are transformed into indexable data objects through the AREI pipeline."

**6. Disclosure language stays in.**
Legal and methodological caveats are not style problems. Keep them. Sharpen everything around them.

**7. No SaaS fluff.**
Avoid: "leverages," "ecosystem," "seamless," "best-in-class," "scalable," "solution," "stakeholders," "empower," "unlock."

**8. No broker language.**
Avoid: "exciting opportunity," "unique proposition," "dynamic market," "key differentiators."

**9. No hype.**
Do not strengthen claims. Do not imply complete coverage, official index status, transaction prices, valuations, or guaranteed accuracy. The product is strong enough without it.

## Do / Don't examples

| Do | Don't |
|---|---|
| "Africa's property data is everywhere. AREI brings it into focus." | "AREI leverages cutting-edge AI to surface actionable market intelligence." |
| "The asset is not the website. The asset is the dataset." | "AREI's proprietary data infrastructure represents a significant competitive moat." |
| "Cape Verde is Market 01. Not the destination." | "Cape Verde serves as our initial proof-of-concept market." |
| "We don't sell properties. We structure the data behind them." | "AREI is uniquely positioned to disrupt the African real estate data ecosystem." |
| "Three surfaces. One engine." | "A single ingestion and methodology engine serves three distinct product surfaces." |
| "Free tools acquire data. Paid intelligence monetizes it." | "Our freemium data acquisition strategy enables professional-tier monetization." |
| "Records that pass go into the index." | "Records that pass the methodology threshold become index-eligible." |
| "Each new market reuses the same parsers, methodology, and publication cadence." | "Each new market reuses the same parser stack, methodology framework, brand system, and publication cadence." |
| "Broker cooperation improves data quality and reduces parser cost." | "Source cooperation accelerates data quality and reduces parser cost." |
| "No incumbent has the incentive to build this. That is the opening." | "There is currently a gap in the market that AREI is uniquely positioned to address." |

## Banned or restricted phrases

**Banned (never use):**

| Phrase | Reason |
|---|---|
| "leverages" | SaaS filler |
| "ecosystem" | overused, abstract |
| "seamless" | meaningless |
| "best-in-class" | hype |
| "unique proposition" | broker language |
| "stakeholders" | corporate filler |
| "empower" / "unlock" | startup cliché |
| "exciting opportunity" | broker language |
| "dynamic market" | meaningless |
| "scalable solution" | SaaS filler |
| "world-class" | unverifiable claim |
| "disrupt" | overused |

**Restricted (use only with care):**

| Phrase | Restriction |
|---|---|
| "scored, source-linked published signal" | Too stacked. Use "clean, sourced market signal" or just "published market signal." |
| "methodology-disclosed" | Legal context only. Elsewhere, "methodology is public" or "with methodology disclosure." |
| "indexable data objects" | Engineering spec only. Not in investor or product copy. |
| "professional and institutional decision-making" | Abstract. Name the actual use: "investors, lenders, and advisors who need to read the market." |
| "data acquisition" | Acceptable as a role label (Broker Tools role: data acquisition). Do not use as a general phrase for "collecting data." |
| "market intelligence layer" | Acceptable in investor context. Do not use in public-facing copy. |
| "coverage scope and confidence score" | Engineering spec. In brief copy: "coverage and confidence score." |

## Surface alignment

The same voice applies across all AREI surfaces. What changes is depth, not register.

| Surface | Depth | Notes |
|---|---|---|
| Landing page | Shortest. Thesis lines only. | The source of the approved phrases above. |
| Investor briefing | Full thesis + supporting detail. | Same voice, more sentences. Disclosure language required. |
| Admin Strategy Check | Direct, internal, no caveats needed. | Same brevity, slightly less formal. |
| Broker materials | Practical, benefits-first. | Same tone. No infrastructure jargon in broker-facing copy. |
| Index publications / briefings | Precise, methodology-disclosed. | Institutional register. Short sentences still apply. |
